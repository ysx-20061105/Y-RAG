# RAG智能知识库系统数据库设计

## 1. 设计目标

- 满足 `specs/001-note-backend-api/spec.md` 的笔记 CRUD 与知识库隔离要求
- 笔记内容以纯 Markdown 存储（`LONGTEXT`），单条不超过 10MB
- 支持 RAG 链路：文档快照 -> 分块 -> 向量映射 -> 检索追踪
- 知识库删除时级联删除关联笔记与RAG元数据
- 支持用户注册/登录：用户身份来自 `user_account`，并通过 JWT 承载用户ID

## 2. 存储分层（推荐）

- 事务与元数据：MySQL 8.x（本设计覆盖）
- 向量检索：独立向量库（Milvus / pgvector / ES 向量引擎）
- MySQL 中只保存向量引用ID与状态，不存放高维向量本体

## 3. ER 关系（MVP + RAG扩展）

- `user_account (1) -> (N) knowledge_base`
- `knowledge_base (1) -> (N) note`
- `note (1) -> (N) note_version`
- `note (1) -> (N) rag_document`
- `rag_document (1) -> (N) rag_chunk`
- `rag_chunk (1) -> (N) rag_chunk_vector_ref`
- `knowledge_base (1) -> (N) rag_retrieval_log`

  description VARCHAR(1000) NULL COMMENT '知识库描述',
  status TINYINT NOT NULL DEFAULT 1 COMMENT '1启用 0禁用',
  created_at DATETIME(3) NOT NULL DEFAULT CURRENT_TIMESTAMP(3),
  updated_at DATETIME(3) NOT NULL DEFAULT CURRENT_TIMESTAMP(3) ON UPDATE CURRENT_TIMESTAMP(3),
  deleted_at DATETIME(3) NULL,

  kb_id BIGINT NOT NULL COMMENT '知识库ID',

  parse_status VARCHAR(32) NOT NULL DEFAULT 'PENDING' COMMENT 'PENDING|RUNNING|SUCCESS|FAILED',

  INDEX idx_vector_status (index_status, updated_at DESC)

CREATE TABLE IF NOT EXISTS rag_retrieval_log (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  kb_id BIGINT NOT NULL,
  actor_user_id BIGINT NOT NULL COMMENT '检索发起用户ID',
  query_text TEXT NOT NULL,
  rewritten_query TEXT NULL,
  top_k INT NOT NULL,
  retrieved_chunk_ids JSON NOT NULL COMMENT '召回chunk id列表',
  latency_ms INT NOT NULL,
  created_at DATETIME(3) NOT NULL DEFAULT CURRENT_TIMESTAMP(3),
  CONSTRAINT fk_retrieval_kb FOREIGN KEY (kb_id) REFERENCES knowledge_base(id) ON DELETE CASCADE,
  CONSTRAINT fk_retrieval_user FOREIGN KEY (actor_user_id) REFERENCES user_account(id) ON DELETE RESTRICT,
  INDEX idx_retrieval_kb_created (kb_id, created_at DESC),
  INDEX idx_retrieval_actor_created (actor_user_id, created_at DESC)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci;

## 5. 与当前需求的映射

- 登录注册能力：由 `user_account` 支撑，令牌会话由 sa-token 管理
- FR-001/004/005/006/007：由 `note` + `knowledge_base` 实现
- FR-002：标题提取逻辑在服务层实现，写入 `note.title`
- FR-003：`note.content` 仅存 Markdown 源文本
- FR-008：`note.kb_id` 外键 + 业务层校验知识库存在性
- FR-009：`created_at`、`updated_at` 自动维护
- FR-010：`content_bytes` + `CHECK` + 应用层双重校验

## 6. 关键索引与查询建议

- 列表页（按知识库分页）：使用 `idx_note_kb_updated (kb_id, updated_at, id)`
- 详情页：主键 `note.id` + 权限校验（`kb_id` 归属当前登录用户）
- 建议使用 Keyset 分页提升稳定性：

```sql
SELECT id, kb_id, title, summary, created_at, updated_at
FROM note
WHERE kb_id = ?
  AND (updated_at < ? OR (updated_at = ? AND id < ?))
ORDER BY updated_at DESC, id DESC
LIMIT ?;
```

## 7. RAG 入库链路（落库顺序）

1. 保存笔记到 `note`
2. 写入 `note_version`（可选，但推荐）
3. 创建 `rag_document(parse_status=PENDING)`
4. 异步分块写 `rag_chunk`
5. 向量化后写 `rag_chunk_vector_ref`（保存 `vector_id`）
6. 检索日志写 `rag_retrieval_log`

## 8. 安全与隔离

- 所有读写接口必须附带用户ID权限过滤
- 推荐查询条件统一为：`kb.owner_user_id = JWT.userId`
- 禁止跨知识库访问：接口必须校验 `note.kb_id` 属于当前登录用户
- 密码仅存 `password_hash`，禁止明文；建议使用 BCrypt/Argon2

## 9. 迁移建议（最小上线集）

第一阶段（必须）：`user_account`、`knowledge_base`、`note`

第二阶段（RAG基础）：`rag_document`、`rag_chunk`、`rag_chunk_vector_ref`

第三阶段（可观测与回溯）：`note_version`、`rag_retrieval_log`

## 10. 表功能与字段说明

### 10.1 `user_account`（用户表）

功能：
- 存储注册用户账号信息
- 支撑登录认证（用户名/邮箱 + 密码哈希）
- 为知识库与操作记录提供用户归属

| 字段 | 类型 | 说明 | 约束/备注 |
|---|---|---|---|
| id | BIGINT | 用户主键 | PK, AUTO_INCREMENT |
| username | VARCHAR(64) | 登录用户名 | NOT NULL, UNIQUE |
| email | VARCHAR(255) | 邮箱 | 可空, UNIQUE |
| password_hash | VARCHAR(255) | 密码哈希 | NOT NULL, 建议 BCrypt/Argon2 |
| nickname | VARCHAR(64) | 昵称 | 可空 |
| avatar_url | VARCHAR(512) | 头像地址 | 可空 |
| status | TINYINT | 账号状态 | 1=正常, 0=禁用 |
| last_login_at | DATETIME(3) | 最近登录时间 | 可空 |
| created_at | DATETIME(3) | 创建时间 | 自动填充 |
| updated_at | DATETIME(3) | 更新时间 | 自动更新 |
| deleted_at | DATETIME(3) | 软删除时间 | 可空 |

### 10.2 `knowledge_base`（知识库表）

功能：
- 作为笔记和RAG数据的逻辑容器
- 与用户一对多绑定，实现数据隔离

| 字段 | 类型 | 说明 | 约束/备注 |
|---|---|---|---|
| id | BIGINT | 知识库主键 | PK, AUTO_INCREMENT |
| owner_user_id | BIGINT | 所属用户ID | NOT NULL, FK -> `user_account.id` |
| name | VARCHAR(255) | 知识库名称 | NOT NULL, 同用户下唯一 |
| description | VARCHAR(1000) | 知识库描述 | 可空 |
| status | TINYINT | 状态 | 1=启用, 0=禁用 |
| created_at | DATETIME(3) | 创建时间 | 自动填充 |
| updated_at | DATETIME(3) | 更新时间 | 自动更新 |
| deleted_at | DATETIME(3) | 软删除时间 | 可空 |

### 10.3 `note`（笔记主表）

功能：
- 存储用户编辑后的纯 Markdown 内容
- 支撑笔记 CRUD、分页列表、详情查询

| 字段 | 类型 | 说明 | 约束/备注 |
|---|---|---|---|
| id | BIGINT | 笔记主键 | PK, AUTO_INCREMENT |
| kb_id | BIGINT | 所属知识库ID | NOT NULL, FK -> `knowledge_base.id` |
| title | VARCHAR(255) | 笔记标题 | NOT NULL（可由H1自动提取） |
| content | LONGTEXT | Markdown正文 | NOT NULL，不存HTML |
| content_bytes | INT | 内容字节数 | NOT NULL，<= 10MB |
| summary | VARCHAR(500) | 摘要 | 可空，可异步生成 |
| created_at | DATETIME(3) | 创建时间 | 自动填充 |
| updated_at | DATETIME(3) | 更新时间 | 自动更新 |

### 10.4 `note_version`（笔记版本表）

功能：
- 记录笔记历史快照，支持追溯/回滚
- 为 RAG 处理提供稳定版本输入

| 字段 | 类型 | 说明 | 约束/备注 |
|---|---|---|---|
| id | BIGINT | 版本记录主键 | PK, AUTO_INCREMENT |
| note_id | BIGINT | 关联笔记ID | NOT NULL, FK -> `note.id` |
| version_no | INT | 版本号 | NOT NULL, 同笔记内唯一 |
| title | VARCHAR(255) | 该版本标题 | NOT NULL |
| content | LONGTEXT | 该版本内容 | NOT NULL |
| content_bytes | INT | 该版本内容字节数 | NOT NULL |
| created_by | BIGINT | 操作人用户ID | NOT NULL, FK -> `user_account.id` |
| created_at | DATETIME(3) | 版本创建时间 | 自动填充 |

### 10.5 `rag_document`（RAG文档任务表）

功能：
- 表示一次“笔记内容 -> RAG处理”的文档任务
- 跟踪解析状态、分块数量、幂等哈希

| 字段 | 类型 | 说明 | 约束/备注 |
|---|---|---|---|
| id | BIGINT | 任务主键 | PK, AUTO_INCREMENT |
| kb_id | BIGINT | 知识库ID | NOT NULL, FK -> `knowledge_base.id` |
| note_id | BIGINT | 来源笔记ID | NOT NULL, FK -> `note.id` |
| source_version_id | BIGINT | 来源版本ID | 可空，指向 `note_version.id` |
| source_type | VARCHAR(32) | 来源类型 | 默认 `NOTE_MD` |
| parse_status | VARCHAR(32) | 解析状态 | `PENDING/RUNNING/SUCCESS/FAILED` |
| chunk_count | INT | 分块数量 | 默认 0 |
| token_count | INT | token总量 | 默认 0 |
| content_hash | CHAR(64) | 内容哈希 | NOT NULL，幂等去重 |
| created_at | DATETIME(3) | 创建时间 | 自动填充 |
| updated_at | DATETIME(3) | 更新时间 | 自动更新 |

### 10.6 `rag_chunk`（RAG分块表）

功能：
- 存储文档切分后的 chunk 文本
- 为向量化和检索提供最小语义单元

| 字段 | 类型 | 说明 | 约束/备注 |
|---|---|---|---|
| id | BIGINT | 分块主键 | PK, AUTO_INCREMENT |
| rag_document_id | BIGINT | 关联文档任务ID | NOT NULL, FK -> `rag_document.id` |
| chunk_no | INT | 分块序号 | NOT NULL, 同文档内唯一 |
| chunk_text | MEDIUMTEXT | 分块文本 | NOT NULL |
| token_count | INT | 分块token数 | NOT NULL |
| start_offset | INT | 起始偏移 | NOT NULL |
| end_offset | INT | 结束偏移 | NOT NULL |
| metadata_json | JSON | 分块元数据 | 可空（标题层级/代码语言等） |
| created_at | DATETIME(3) | 创建时间 | 自动填充 |

### 10.7 `rag_chunk_vector_ref`（向量引用表）

功能：
- 管理 chunk 与向量库记录的映射
- 解耦 MySQL 元数据与外部向量引擎

| 字段 | 类型 | 说明 | 约束/备注 |
|---|---|---|---|
| id | BIGINT | 映射主键 | PK, AUTO_INCREMENT |
| chunk_id | BIGINT | 关联分块ID | NOT NULL, FK -> `rag_chunk.id` |
| embedding_model | VARCHAR(128) | 向量模型名 | NOT NULL |
| embedding_dim | INT | 向量维度 | NOT NULL |
| vector_store | VARCHAR(32) | 向量库类型 | NOT NULL（milvus/pgvector/es） |
| vector_id | VARCHAR(128) | 向量记录ID | NOT NULL，外部主键 |
| index_status | VARCHAR(32) | 建索引状态 | `PENDING/SUCCESS/FAILED` |
| created_at | DATETIME(3) | 创建时间 | 自动填充 |
| updated_at | DATETIME(3) | 更新时间 | 自动更新 |

### 10.8 `rag_retrieval_log`（检索日志表）

功能：
- 记录RAG召回请求与性能指标
- 支撑可观测性、质量分析、问题追踪

| 字段 | 类型 | 说明 | 约束/备注 |
|---|---|---|---|
| id | BIGINT | 日志主键 | PK, AUTO_INCREMENT |
| kb_id | BIGINT | 知识库ID | NOT NULL, FK -> `knowledge_base.id` |
| actor_user_id | BIGINT | 发起检索用户ID | NOT NULL, FK -> `user_account.id` |
| query_text | TEXT | 原始查询 | NOT NULL |
| rewritten_query | TEXT | 改写后查询 | 可空 |
| top_k | INT | 召回条数 | NOT NULL |
| retrieved_chunk_ids | JSON | 召回chunk列表 | NOT NULL |
| latency_ms | INT | 端到端耗时(ms) | NOT NULL |
| created_at | DATETIME(3) | 创建时间 | 自动填充 |
