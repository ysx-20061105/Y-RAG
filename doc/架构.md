#### 3. RAG 核心服务实现 (`RagService.java`)

这是系统的“大脑”，负责将 Markdown 笔记转化为向量，并根据用户问题进行检索。

```java
@Service
public class RagService {

    private final VectorStore vectorStore;
    private final ChatClient chatClient;
    private final NoteRepository noteRepository;

    // 构造函数注入
    public RagService(VectorStore vectorStore, ChatClient.Builder chatClientBuilder, NoteRepository noteRepository) {
        this.vectorStore = vectorStore;
        this.chatClient = chatClientBuilder.build();
        this.noteRepository = noteRepository;
    }

    /**
     * 1. 索引流程：当笔记保存时调用
     * 将 Markdown 内容分块并存入 Qdrant
     */
    @Transactional
    public void indexNote(Note note) {
        // 1. 准备文档
        // 使用 Spring AI 的 Document 对象，Metadata 用于过滤（例如限定在特定知识库）
        Document document = new Document(
            note.getId().toString(), // ID
            note.getContent(),       // 内容
            Map.of(
                "kbId", note.getKnowledgeBaseId().toString(),
                "title", note.getTitle()
            )
        );

        // 2. 写入向量库 (Spring AI 会自动处理 Embedding 和分块)
        // 注意：生产环境建议使用异步或消息队列处理此步骤，避免阻塞保存接口
        vectorStore.add(List.of(document));
    }

    /**
     * 2. 检索与问答流程
     * @param query 用户问题
     * @param kbId 知识库ID (用于过滤)
     * @return Flux<String> 流式响应内容
     */
    public Flux<String> retrieveAndChat(String query, Long kbId) {
        
        // A. 构建查询对象，添加过滤条件
        Query searchQuery = Query.builder()
                .query(query)
                .filterExpression(String.format("kbId == '%s'", kbId)) // Qdrant 过滤语法
                .topK(5) // 检索最相关的 5 个片段
                .build();

        // B. 执行相似度搜索
        List<Document> relevantDocs = vectorStore.similaritySearch(searchQuery);

        // C. 构建上下文信息
        String context = relevantDocs.stream()
                .map(Document::getContent)
                .collect(Collectors.joining("\n---\n"));

        // D. 组装提示词 (Prompt Engineering)
        String systemPrompt = """
            你是一个专业的知识库助手。
            请根据以下参考资料回答用户的问题。
            如果参考资料中没有相关信息，请说明你不知道。
            
            参考资料：
            %s
            """;

        // E. 调用大模型 (流式)
        Prompt prompt = new Prompt(
            systemPrompt.formatted(context), 
            new UserMessage(query)
        );

        // 返回流式数据，让 Controller 直接透传给前端
        return chatClient.prompt(prompt)
                .stream()
                .content();
    }
}
```

#### 4. 控制器层 (`ChatController.java`)

利用 JDK 21 的虚拟线程特性（`@RequestAspect` 或配置 Tomcat 使用虚拟线程），可以高效处理 SSE。

```java
@RestController
@RequestMapping("/api/chat")
public class ChatController {

    private final RagService ragService;

    public ChatController(RagService ragService) {
        this.ragService = ragService;
    }

    @PostMapping(value = "/stream", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public Flux<ServerSentEvent<String>> chatStream(@RequestBody ChatRequest request) {
        // 调用 Service 获取流
        return ragService.retrieveAndChat(request.query(), request.kbId())
                .map(content -> ServerSentEvent.builder(content).build())
                .onErrorResume(e -> {
                    // 错误处理
                    return Flux.just(ServerSentEvent.builder("Error: " + e.getMessage()).event("error").build());
                });
    }
}
```

### 💻 前端架构 (Vue 3)

前端保持不变，重点在于适配 SSE 流式接收。

#### 核心逻辑 (`useChat.ts`)

```typescript
import { ref } from 'vue';

export function useChat() {
  const messages = ref<any[]>([]);
  const isStreaming = ref(false);

  const sendMessage = async (query: string, kbId: number) => {
    // 添加用户消息
    messages.value.push({ role: 'user', content: query });
    
    // 创建空的 AI 消息占位
    const aiMessage = { role: 'assistant', content: '' };
    messages.value.push(aiMessage);
    
    isStreaming.value = true;

    try {
      const response = await fetch('/api/chat/stream', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ query, kbId })
      });

      const reader = response.body?.getReader();
      const decoder = new TextDecoder();

      while (true) {
        const { done, value } = await reader!.read();
        if (done) break;

        const chunk = decoder.decode(value);
        // 解析 SSE 格式 (data: ...)
        const lines = chunk.split('\n');
        lines.forEach(line => {
          if (line.startsWith('data: ')) {
            const content = line.replace('data: ', '');
            aiMessage.content += content; // 累加内容
          }
        });
      }
    } finally {
      isStreaming.value = false;
    }
  };

  return { messages, sendMessage, isStreaming };
}
```

### 🚀 为什么选择这个架构？

1.  **JDK 21 + Spring Boot 3.4**: 你的应用将能够以极低的资源消耗处理数千个并发的 WebSocket/SSE 连接，这对于多人同时使用 RAG 系统至关重要。
2.  **Qdrant**: 相比 Milvus，Qdrant 更轻量（单二进制文件），且对过滤查询（Filtering）的支持非常强大，这对于“指定知识库聊天”这个需求非常关键（`kbId` 过滤）。
3.  **Spring AI Alibaba**: 官方支持的组件，比手写 HTTP Client 调用通义千问更稳定，且自动处理了 Token 计数、重试等细节。

你可以直接按照这个结构开始搭建项目，先跑通 `RagService` 中的向量化流程，再对接前端。
