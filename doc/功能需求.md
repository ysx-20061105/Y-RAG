### 功能需求文档：Markdown编辑器与实时渲染模块

#### 文档信息
- **文档版本:** V1.0
- **模块名称:** 笔记编辑器
- **技术路线:** 纯前端渲染
- **最后更新日期:** 2026-03-30

#### 模块概述
本模块是RAG知识库系统的内容生产核心。旨在为用户提供一个零延迟、高隐私、体验流畅的Markdown编辑环境。通过纯前端技术栈，实现用户输入与预览渲染的毫秒级同步，确保“所见即所得”的写作体验。同时，确保输出的Markdown源数据纯净、结构化，为后续的RAG检索与处理流程提供高质量输入。

#### 用户故事
- **作为** 一名知识工作者，**我希望** 在输入Markdown语法的瞬间就能看到格式化后的效果，**以便** 我能专注于内容创作本身，而不是记忆和检查语法符号。
- **作为** 一名技术人员，**我希望** 编辑器能自动识别并为代码块进行语法高亮，**以便** 我能清晰、美观地展示代码片段。
- **作为** 一名隐私敏感用户，**我希望** 我的笔记内容在编辑过程中完全保留在本地浏览器，**以便** 确保我的知识资产在保存前不会泄露。

#### 功能需求详情
**编辑器核心交互**
- **分屏编辑模式**
    - 界面需默认采用左右分栏布局。左侧为“源码编辑区”，右侧为“实时预览区”。
    - 支持用户拖拽中间的分隔线，以调整左右两栏的宽度比例。
    - 提供快捷按钮，支持一键切换至“专注模式”（仅显示编辑区）或“预览模式”（仅显示预览区）。
- **实时渲染机制**
    - **零延迟同步:** 用户在编辑区的任何输入、删除、粘贴操作，必须在 **50ms** 内触发右侧预览区的渲染更新。
    - **滚动同步:** 当用户在编辑区滚动时，预览区需根据内容比例自动、平滑地同步滚动位置，保持视觉上下文的一致性。
- **Markdown语法支持**
    - **基础语法:** 必须完美支持标题(H1-H6)、粗体、斜体、删除线、引用、无序/有序列表、分割线。
    - **高级语法:** 必须支持表格（含对齐方式）、脚注、任务列表、数学公式 (LaTeX/KaTeX)。
    - **代码块:** 支持fenced code blocks (```)，并能自动识别常见的编程语言（如Python, JavaScript, Java, Go, SQL等）。

**纯前端渲染特性**
- **本地解析引擎**
    - 系统需集成一个成熟的轻量级前端Markdown解析库（例如 `markdown-it` 或 `marked`）。
    - 所有从Markdown文本到HTML的转换过程必须在用户的浏览器中完成，严禁在编辑过程中向服务器发送任何渲染请求。
- **语法高亮**
    - 需集成一个前端代码高亮库（例如 `highlight.js` 或 `Prism.js`）。
    - 预览区中的代码块必须自动应用预设的配色主题，并在代码块的右上角显示当前语言类型的标签。
- **图片处理**
    - **拖拽上传:** 支持用户将本地图片文件直接拖拽到编辑器中。编辑器应自动触发上传流程，并在上传成功后，在光标位置插入标准的Markdown图片链接语法 `![alt](url)`。
    - **预览渲染:** 预览区需正确加载并显示图片。支持点击图片后，在模态框中以原始尺寸放大预览。

**数据输出与RAG适配**
- **纯净源数据**
    - 当用户触发“保存”操作时，系统需从编辑区提取未经任何修改的原始Markdown文本。
    - 在将此文本发送给后端进行RAG处理（如分块、向量化）之前，必须确保其是纯净的Markdown格式，不包含任何前端渲染时附加的HTML标签或DOM属性。
- **元数据提取**
    - 前端需具备自动解析能力，将Markdown内容的第一行标题（例如 `# 笔记标题`）自动提取出来，作为笔记的默认名称，除非用户已手动输入了自定义名称。

#### 非功能性需求
**性能指标**
- **渲染延迟:** 在编辑一篇5000字的长文档时，从输入到完成渲染的端到端延迟不得超过 **200ms**。
- **资源占用:** 前端解析库和高亮库的打包体积总和需经过优化，避免对首屏加载时间造成显著影响。

**兼容性**
- 必须支持所有主流现代浏览器，包括最新版本的 Chrome, Edge, Safari, Firefox。
- 在移动端或窄屏设备上，界面需自动响应，切换为单栏编辑模式，并将实时预览功能隐藏或移至底部，通过一个“预览”按钮来触发。

**安全性**
- **XSS防护:** 前端Markdown渲染引擎必须开启HTML过滤或转义功能，以防止用户输入恶意脚本（如`<script>`标签）并在预览区执行，从而避免跨站脚本攻击。

#### 界面原型描述
- **顶部工具栏:** 包含“加粗”、“斜体”、“标题”、“插入链接”、“插入代码”、“上传图片”等快捷按钮。点击任一按钮，应在编辑器光标位置插入对应的Markdown语法。
- **左侧编辑区:** 采用深色背景、等宽字体，并显示行号，以提供舒适的代码编写体验。
- **右侧预览区:** 采用浅色/深色背景（跟随系统设置）、衬线字体。CSS样式需针对阅读进行优化，例如设置合适的行高、段间距和标题大小。
- **底部状态栏:** 显示当前笔记的字数统计、光标所在的行/列位置，以及一个“已同步”或“保存成功”的状态指示。

#### 验收标准
- 在编辑区输入 `# Hello` 并按下回车，右侧预览区应立即显示为大号、加粗的“Hello”文本。
- 在编辑区输入 ```python 和一个简单的Python代码，右侧预览区应立即显示为一个带有Python语法高亮的代码块，并标注“python”标签。
- 断开网络连接后，用户依然可以在编辑器中正常输入、删除，并且右侧预览区能正常渲染。
- 保存笔记后，检查数据库中存储的内容，必须为纯文本的Markdown格式，不包含任何HTML标签。
