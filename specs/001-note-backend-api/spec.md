# Feature Specification: RAG系统笔记后端接口

**Feature Branch**: `001-note-backend-api`
**Created**: 2026-03-30
**Status**: Draft
**Input**: 完成本RAG系统的笔记功能，参考doc/功能需求.md，现只实现后端接口

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 笔记创建与保存 (Priority: P1)

用户能够在知识库中创建新笔记，并保存纯Markdown格式的笔记内容。

**Why this priority**: 笔记创建是核心功能，没有笔记就没有后续的RAG检索和问答。

**Independent Test**: 可以独立测试：创建笔记 -> 保存 -> 查询笔记内容是否一致

**Acceptance Scenarios**:

1. **Given** 用户已登录且拥有知识库权限，**When** 创建一个新笔记并填充Markdown内容，**Then** 笔记成功保存，数据库中存储的是纯Markdown文本，不包含任何HTML标签
2. **Given** 用户创建笔记时未指定标题，**When** 系统自动从内容的第一行H1提取标题，**Then** 笔记标题自动设置为该H1文本
3. **Given** 用户已有一个笔记，**When** 更新笔记内容后保存，**Then** 更新后的内容正确持久化，旧版本被覆盖

---

### User Story 2 - 笔记查询与列表 (Priority: P1)

用户能够查询指定知识库下的所有笔记列表，并能够查看单个笔记的详细内容。

**Why this priority**: 查询和列表是用户访问已有笔记的主要方式。

**Independent Test**: 可以独立测试：创建多个笔记 -> 分页查询列表 -> 查看单个笔记详情

**Acceptance Scenarios**:

1. **Given** 用户拥有知识库访问权限，**When** 请求该知识库下的笔记列表（分页），**Then** 返回笔记列表，包含每条笔记的ID、标题、摘要、创建时间、更新时间
2. **Given** 用户知道笔记ID，**When** 请求获取笔记详情，**Then** 返回完整笔记内容（纯Markdown格式）
3. **Given** 用户未提供有效的笔记ID，**When** 请求笔记详情，**Then** 返回404错误，并提示笔记不存在

---

### User Story 3 - 笔记删除 (Priority: P2)

用户能够删除不需要的笔记。

**Why this priority**: 删除功能虽不常用，但必须有完整的CRUD支持。

**Independent Test**: 可以独立测试：创建笔记 -> 删除笔记 -> 尝试查询该笔记

**Acceptance Scenarios**:

1. **Given** 用户拥有笔记所在知识库的删除权限，**When** 删除一个存在的笔记，**Then** 笔记从数据库中移除，后续查询返回404
2. **Given** 用户尝试删除一个不存在的笔记，**Then** 返回404错误，不影响系统状态

---

### User Story 4 - 笔记与知识库关联 (Priority: P1)

每个笔记必须属于一个知识库，支持按知识库过滤笔记。

**Why this priority**: 知识库是笔记的容器，RAG检索需要限定在特定知识库范围内。

**Independent Test**: 可以独立测试：创建不同知识库下的笔记 -> 按知识库ID查询 -> 验证笔记仅返回属于该知识库的结果

**Acceptance Scenarios**:

1. **Given** 笔记创建时指定了知识库ID，**When** 保存笔记，**Then** 笔记与该知识库建立关联关系
2. **Given** 用户查询某知识库的笔记列表，**When** 执行过滤查询，**Then** 仅返回属于该知识库的笔记，不返回其他知识库的笔记

---

### Edge Cases

- ~~当知识库被删除时，该知识库下的所有笔记是否同步删除？~~ → 已确认：级联删除
- ~~当用户尝试访问不属于其权限范围内的笔记时，系统如何处理？~~ → 已确认：严格隔离
- ~~当笔记内容为空时，系统应允许保存空笔记还是返回验证错误？~~ → 已确认：允许保存，标题默认"无标题笔记"
- ~~当笔记内容过长时，系统是否需要限制大小？~~ → 已确认：限制10MB，实时提示用户剩余容量

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统 MUST 提供笔记创建接口，允许用户提交Markdown内容并关联到指定知识库
- **FR-002**: 系统 MUST 在保存笔记时自动提取内容第一行H1作为笔记标题（如果用户未手动指定）
- **FR-003**: 系统 MUST 以纯Markdown格式存储笔记内容，不添加任何HTML标签或渲染产物
- **FR-004**: 系统 MUST 提供笔记列表查询接口，支持按知识库ID分页查询
- **FR-005**: 系统 MUST 提供笔记详情查询接口，返回完整的纯Markdown内容
- **FR-006**: 系统 MUST 提供笔记更新接口，支持修改内容和标题
- **FR-007**: 系统 MUST 提供笔记删除接口，支持物理删除
- **FR-008**: 系统 MUST 验证笔记所属知识库的存在性，无效知识库ID应返回错误
- **FR-009**: 系统 MUST 记录笔记的创建时间和最后更新时间
- **FR-010**: 系统 MUST 限制单条笔记内容大小不超过10MB，并在内容接近限制时实时提示用户剩余容量

### Key Entities *(include if feature involves data)*

- **笔记 (Note)**: 代表用户的单个Markdown笔记，包含唯一标识、关联知识库ID、标题、内容、创建时间、更新时间
- **知识库 (KnowledgeBase)**: 笔记的容器/分组，同一知识库下的笔记会一起参与RAG检索

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 用户能在3秒内完成笔记创建并收到成功响应
- **SC-002**: 笔记列表查询响应时间不超过500ms（100条记录以内）
- **SC-003**: 笔记保存后，从数据库直接读取的内容与提交内容完全一致，无格式损坏
- **SC-004**: 笔记查询时，未找到返回404且响应体包含明确错误信息
- **SC-005**: 系统能正确区分和过滤不同知识库下的笔记

## Clarifications

### Session 2026-03-30

- Q: 当知识库被删除时，该知识库下的所有笔记是否同步删除？ → A: 级联删除（知识库删除时自动删除所有关联笔记）
- Q: 当用户尝试访问不属于其权限范围内的笔记时，系统如何处理？ → A: 严格隔离（用户只能访问其所属知识库下的笔记）
- Q: 当笔记内容为空时，系统应允许保存空笔记还是返回验证错误？ → A: 允许保存但标题使用默认值"无标题笔记"
- Q: 当笔记内容过长时，系统是否需要限制大小？ → A: 限制10MB，实时提示用户剩余容量

## Assumptions

- 用户已通过JWT认证，系统能从中提取用户ID和权限信息
- 知识库管理接口已存在或同步开发，笔记接口依赖知识库ID的有效性
- 前端会负责Markdown内容的实时渲染，后端只负责存储纯净的Markdown文本
- 删除笔记时同步删除该笔记在向量数据库中的关联向量数据（由RAG服务处理）
- 使用MySQL作为关系型数据库存储笔记和知识库数据
- 笔记内容的字符编码为UTF-8
